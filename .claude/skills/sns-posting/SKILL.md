# SNS Posting

플랫폼별 특성에 맞는 SNS 포스팅 작성 스킬.

## 핵심 원칙

### 1. One Idea Per Post
- 하나의 포스팅 = 하나의 메시지
- 여러 아이디어를 욱여넣지 않기

### 2. Authenticity Over Performance
- 가짜 engagement bait 금지 ("Agree?", "Follow me for more")
- 진짜 경험, 진짜 인사이트
- 과장된 성공 스토리보다 솔직한 과정

### 3. 플랫폼별 최적화
- 같은 내용이라도 플랫폼마다 톤/길이/포맷이 다름
- 복붙 금지, 각 플랫폼에 맞게 재가공

## 플랫폼 가이드

| 플랫폼 | 특징 | 상세 |
|--------|------|------|
| LinkedIn | 전문성 + 스토리텔링 | [linkedin.md](references/linkedin.md) |
| Threads | 짧고 캐주얼 | [threads.md](references/threads.md) |
| X/Twitter | 임팩트 + 스레드 | [twitter.md](references/twitter.md) |

## 포스팅 구조 (공통)

```
1. Hook (첫 줄)
   - 스크롤 멈추게 하는 한 줄
   - 숫자, 질문, 반전, 선언

2. Story/Body
   - 개인 경험 또는 인사이트
   - 짧은 문장, 충분한 줄바꿈

3. CTA (마무리)
   - 링크, 질문, 또는 깔끔한 마무리
   - 억지 engagement 유도 X
```

## Anti-patterns (피해야 할 것)

- "This one thing made me $X" 류의 뻔한 훅
- "저는 실패했습니다... (사실 성공 자랑)" 가짜 겸손
- "매일 새벽 4시에 일어납니다" 허세
- "🔥🚀💯" 이모지 과다
- "Agree? Comment below!" 가짜 참여 유도
- 모든 문장에 줄바꿈 (가독성 해침)

## 작성 프로세스

1. **플랫폼 확인**: 어디에 올릴 건지
2. **핵심 메시지 정의**: 딱 하나만
3. **Hook 작성**: 첫 줄에 집중
4. **본문 전개**: 경험/인사이트 녹이기
5. **플랫폼별 조정**: 길이, 톤, 포맷 맞추기
6. **Anti-pattern 체크**: 위 목록 확인

## 톤 가이드

```
More polished than: 카톡 메시지
Less formal than: 회사 공식 보도자료

= 똑똑한 동료와 커피챗하는 느낌
```
